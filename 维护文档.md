## 二、服务管理（PM2）

```bash
# 查看状态
sudo pm2 status

# 查看实时日志
sudo pm2 logs cert-manager

# 查看最近 100 行日志

sudo pm2 restart cert-manager --update-env

# 停止服务
sudo pm2 stop cert-manager

# 启动服务
sudo pm2 start cert-manager
```

## 三、更新后端代码

### 方式一：更新单个文件

```bash
# 在本机终端执行
scp -P 23 /本地/文件路径/xxx.js zhiwen@125.110.207.231:/tmp/

# 然后在服务器上
echo "123" | sudo -S cp /tmp/xxx.js /www/wwwroot/cert-manager/deploy/src/对应目录/xxx.js
sudo pm2 restart cert-manager --update-env
```

### 方式二：更新整个 src 目录

**在本机执行：**
```bash
# 打包（排除 .DS_Store）
cd /你的项目目录
tar --exclude='*.DS_Store' -czf /tmp/cert-src-new.tar.gz src/

# 上传并部署
scp -P 23 /tmp/cert-src-new.tar.gz zhiwen@125.110.207.231:/tmp/
ssh -p 23 zhiwen@125.110.207.231 'echo "123" | sudo -S tar -xzf /tmp/cert-src-new.tar.gz -C /www/wwwroot/cert-manager/deploy/ && echo "123" | sudo -S pm2 restart cert-manager --update-env && echo "部署完成"'
```

### 方式三：更新完整项目（含前端）

**在本机执行：**
```bash
# 1. 打包后端
cd /你的后端目录
tar --exclude='node_modules' --exclude='.env' --exclude='*.DS_Store' -czf /tmp/cert-deploy.tar.gz .

# 2. 打包前端（或使用已有 dist.zip）
cd /你的前端目录
npm run build
zip -r /tmp/dist.zip dist/

# 3. 上传
scp -P 23 /tmp/cert-deploy.tar.gz /tmp/dist.zip zhiwen@125.110.207.231:/tmp/

# 4. 在服务器上部署
ssh -p 23 zhiwen@125.110.207.231
```

**在服务器上执行：**
```bash
# 解压后端（保留 .env 和 data 目录）
echo "123" | sudo -S tar -xzf /tmp/cert-deploy.tar.gz -C /www/wwwroot/cert-manager/deploy/ --exclude='.env'

# 解压前端
sudo mkdir -p /www/wwwroot/cert-manager/deploy/client
unzip -o /tmp/dist.zip -d /tmp/dist_tmp
sudo cp -r /tmp/dist_tmp/dist/. /www/wwwroot/cert-manager/deploy/client/
rm -rf /tmp/dist_tmp

# 安装新依赖（如果 package.json 有变化）
cd /www/wwwroot/cert-manager/deploy && sudo npm install --production

# 重启
sudo pm2 restart cert-manager --update-env
```

## 六、数据库管理

### 连接数据库
```bash
PGPASSWORD='CertManager@2025' psql -U CertManager -h 127.0.0.1 -d CertManager
```

### 修改数据库密码
```bash
sudo -u postgres psql
```
```sql
ALTER USER "CertManager" WITH PASSWORD '新密码';
\q
```
然后同步修改 `.env` 里的 `PG_PASSWORD`。

### 修改数据库名
```sql
-- 需要先断开所有连接
sudo pm2 stop cert-manager
ALTER DATABASE "CertManager" RENAME TO "新数据库名";
\q
```
然后修改 `.env` 里的 `PG_DATABASE`，再重启。

### 手动备份数据库
```bash
PGPASSWORD='CertManager@2025' pg_dump -U CertManager -h 127.0.0.1 CertManager > ~/backup_$(date +%Y%m%d).sql
```

### 恢复数据库
```bash
PGPASSWORD='CertManager@2025' psql -U CertManager -h 127.0.0.1 CertManager < ~/backup_20260328.sql
```

### 添加缺失的数据库列（手动 migration）
```bash
PGPASSWORD='CertManager@2025' psql -U CertManager -h 127.0.0.1 -d CertManager -c \
  "ALTER TABLE 表名 ADD COLUMN IF NOT EXISTS 列名 类型;"
```

## 八、SSL 证书配置

```bash
# 安装 certbot
sudo apt install -y certbot python3-certbot-nginx

# 申请证书（所有域名）
sudo certbot --nginx \
  -d p12.g600.cn \
  -d p12.6ta.net \
  -d p12.6wz.net \
  -d p12.lla8.com \
  -d p12.52ojk.com \
  -d p12.kefu.me

# 查看证书状态
sudo certbot certificates

# 手动续期
sudo certbot renew
```

```bash

# 测试连接
PGPASSWORD='CertManager@2025' psql -U CertManager -h 127.0.0.1 -d CertManager -c "SELECT 1;"
```

### 前端 404
```bash
# 检查前端文件
ls /www/wwwroot/cert-manager/deploy/client/
# 应该有 index.html 和 assets 目录
```
